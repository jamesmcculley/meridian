# Lab 05: Policy-As-Code

This lab adds lightweight checks that protect the lab from unsafe configuration
changes.

## Objective

Create narrow, understandable policy checks for repository artifacts such as
network rules, topology files, or workflow configuration.

## Current State

Scaffold only. The first real GitHub Actions policy check is intentionally left
as a hands-on learning task.

## Expected Evidence

When implemented, this lab should show:

- one clearly scoped policy rule
- one accepted example
- one rejected example
- CI or local validation output
- documented limits of the policy

See `LEARNING_TASK.md` for the implementation prompt.
