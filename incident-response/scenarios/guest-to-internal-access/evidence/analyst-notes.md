# Analyst Notes

Scenario: guest-to-internal-access.

## Initial Assessment

The event `evt-denied-guest-access-001` shows `branch-guest-client` traffic
attempting to reach `branch-local-service`. The event action is `deny`, and the
flow ID is `D2`.

## Why This Matters

Guest access to branch services is a defined trust-boundary violation attempt.
The expected control behavior is denial by default. The current sample confirms
that the event can be validated, mapped to the deny flow, and reported through
`meridian-detect`.

## Analyst Determination

- Classification: expected denial of guest-to-internal access.
- Severity: medium.
- Immediate containment: none for repository sample data.
- Follow-up: preserve the deny rule, add runtime evidence when topology exists,
  and enrich future events with recurrence and source instance context.
